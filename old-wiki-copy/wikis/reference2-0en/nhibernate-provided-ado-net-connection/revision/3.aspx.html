NHibernate provided ADO.NET connection

Alternatively, you can have the ISessionFactory open connections for you. The ISessionFactory must be provided with ADO.NET connection properties in one of the following ways:

  1. Pass an instance of IDictionary<string, string> (property name, property value) to Configuration.SetProperties(IDictionary<string, string>).

  2. Include <property> elements in a configuration section in the application configuration file. The section should be named hibernate-configuration and its handler set to NHibernate.Cfg.ConfigurationSectionHandler. The XML namespace of the section should be set to urn:nhibernate-configuration-2.2.

  3. Include <property> elements in hibernate.cfg.xml (discussed later).

If you take this approach, opening an ISession is as simple as: 

ISession session = sessions.OpenSession(); // open a new Session
/* do some data access work, an ADO.NET connection will be used on demand */

All NHibernate property names and semantics are defined on the class NHibernate.Cfg.Environment. We will now describe the most important settings for ADO.NET connection configuration.

NHibernate will obtain (and pool) connections using an ADO.NET data provider if you set the following properties:

NHibernate ADO.NET Properties

<tbody> </tbody>
Property name Purpose
connection.provider_class

The type of a custom <tt>IConnectionProvider</tt>.

eg. <tt>full.classname.of.ConnectionProvider</tt> if the Provider is built into NHibernate, or <tt>full.classname.of.ConnectionProvider, assembly</tt> if using an implementation of <tt>IConnectionProvider</tt> not included in NHibernate.
connection.driver_class

The type of a custom <tt>IDriver</tt>, if using <tt>DriverConnectionProvider</tt>.

<tt>full.classname.of.Driver</tt> if the driver is built into NHibernate, or <tt>full.classname.of.Driver, assembly</tt> if using an implementation of IDriver not included in NHibernate.

This is usually not needed, most of the time the <tt>dialect</tt> will take care of setting the <tt>IDriver</tt> using a sensible default. See the API documentation of the specific dialect for the defaults.
connection.connection_string Connection string to use to obtain the connection.
connection.connection_string_name The name of the connection string (defined in <tt><connectionStrings> </tt>configuration file element) to use to obtain the connection.
connection.isolation

Set the ADO.NET transaction isolation level. Check <tt>System.Data.IsolationLevel</tt> for meaningful values and the database's documentation to ensure that level is supported.

eg. <tt>Chaos, ReadCommitted, ReadUncommitted, RepeatableRead, Serializable, Unspecified</tt>
connection.release_mode

Specify when NHibernate should release ADO.NET connections. See Connection Release Modes.

eg. <tt>auto</tt> (default) | <tt>on_close</tt> | <tt>after_transaction</tt>

Note that this setting only affects <tt>ISession</tt>s returned from <tt>ISessionFactory.OpenSession</tt>. For <tt>ISession</tt>s obtained through <tt>ISessionFactory.GetCurrentSession</tt>, the <tt>ICurrentSessionContext</tt> implementation configured for use controls the connection release mode for those <tt>ISession</tt>s. See Contextual Sessions.
command_timeout Specify the default timeout of <tt>IDbCommands</tt> generated by NHibernate.
adonet.batch_size Specify the batch size to use when batching update statements. Setting this to 0 (the default) disables the functionality. See Batch updates.
