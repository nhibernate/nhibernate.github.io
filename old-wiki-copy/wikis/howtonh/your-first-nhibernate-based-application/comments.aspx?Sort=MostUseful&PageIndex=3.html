For those who are asking about VS 2008 and are using the built-in VS unit test framework.

You must provide the attribute DeploymentItem for each test method and for each file required to be in the output folder. You can use an absolute path or a relative but consider that it must relative to the solution folder. In my example, my solution is located in a folder NHibernateTester, my application project folder in NHibernateTester\NHibernateTester and my unit test project folder in NHibernateTester\UnitTests

Example:

       [TestMethod]

       [DeploymentItem(@"NHibernateTester\NHibernateTester.sdf")]

       [DeploymentItem(@"UnitTests\hibernate.cfg.xml")]

       public void TestCanUpdateExistingProduct()

       {

           Product product = Products[0];

           product.Name = "Yellow Pear";

           IProductRepository repository = new ProductRepository();

           repository.Update(product);

           // use session to try to load the product

           using (ISession session = _sessionFactory.OpenSession())

           {

               Product fromDb = session.Get<Product>(product.Id);

               Assert.AreEqual(product.Name, fromDb.Name);

           }

       }
