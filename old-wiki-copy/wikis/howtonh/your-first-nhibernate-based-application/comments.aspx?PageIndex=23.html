By: Spooky Posted on 01-03-2012 8:15

Hi

first of all I want to thank Gabriel Schenker for that great post. I am pretty new to NHibernate and it helped me a lot to get a quick overview how to create a first NHibernate bases application.

But despite all, there are a few things which need to be updated otherwise it will not work. Coni2k already mentioned a few “bugs” so I hope it will help him and other visitors:

- “First Sample” is only mentioned once. So I think that’s not really a problem ;-)

- The SharedLibs folder has to be copied into the “FirstSolution” folder (like “..\FirstSolution\FirstSolution\SharedLibs”)

- The “Post-build event command line” command will only work if the complete path to the dll has no blanks!!! If you have folders with blanks (like I do) for example you have to use direct path with double quotes like copy "D:\_PRIVAT\05 Projekte\FirstSolution\FirstSolution\SharedLibs\sqlce*.dll" "D:\_PRIVAT\05 Projekte\FirstSolution\FirstSolution\bin\Debug\"

- To get no errors when creating “NHibernateHelper.cs” class, the lib NHibernate.dll has to be added to the References of the “FirstSolution” project

- It is true that SchemaExport only has three arguments. Just use SchemaExport(cfg).Execute(false, true, false) instead

- As you noticed, you get an error that product table does not exist if you start tests. Please douple check, you have not forgotten to set attribute [SetUp] right before method “SetupContext()” in class “ProductRepository_Fixture”. I forgot it by myself and it took me one hour to find that problem *ARGGGG*

I hope I could help everybody who follows that guide.
